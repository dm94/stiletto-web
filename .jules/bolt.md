## 2025-07-15 - [O(N^2) Recursive Tree Traversals & Spreads]
**Learning:** In deeply nested structures like the Last Oasis tech tree containing hundreds of items, performing nested array-scanning filter searches (`items.filter(...)`) for every child node creation degrades tree building performance to $O(N^2)$ complexity. Similarly, flattening the tree recursively using standard array-spreading (`[...acc, ...children]`) creates significant memory churn and additional $O(N^2)$ array cloning overhead.
**Action:** Pre-group array data into an $O(N)$ indexed `Map` or hash lookup (e.g. grouped by parent id) via React's `useMemo` before rendering or constructing the tree structure, and pass a mutable accumulator array down recursion paths to accumulate tree nodes in-place in linear time.
